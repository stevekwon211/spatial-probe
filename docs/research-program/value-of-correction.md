## Motivation
The precise gap: AV labeling budgets are finite, and the field's default triage is ERROR-COUNT — find the noisiest labels and fix those first (label-quality audits, geometric box-correction like the WACV-2026 line, benchmark-anchors.md:159). But the quantity a practitioner actually wants to maximize per correction dollar is DOWNSTREAM MODEL MOVEMENT, and there is no public benchmark that asks 'which corrections move the model most' as opposed to 'which labels are most wrong'. These two orderings are assumed identical and are almost certainly not: a large box error on a distant, rare, or redundant object can move a trained detector LESS than a small error on a frequent, decision-relevant one — the same way occquery found that a static geometric fact (0.8 m free-width) is NOT the same as the danger verdict it was mistaken for (h3-real-data-findings.md:189-201). The data-valuation literature (Shapley/influence/OpenDataVal) values TRAINING POINTS for a model but is rarely tied to a CORRECTION BUDGET on a real 3D-perception task with a real downstream metric. The contribution is to DEFINE that axis on a public substrate (Argoverse 2 detection): a released {budget × downstream-recovery} benchmark + a value ranker, with the falsifiable claim that value-ranked correction beats both random and error-count at equal budget. Why it fits the lab: it is the 'valuation' axis (PLAN.md L35), a DEFINE-category paper whose bar is 'public data + measurable gap vs a public baseline + a released benchmark' (benchmark-anchors.md:13-16,55) — exactly the lab's strength (occquery's beat-random/expressivity discipline transposed to beat-random/beat-error-count). Why later, not now: it needs a trainable detector and a retrain loop (GPU), neither present in the repo nor runnable on this Mac, so it is correctly sequenced AFTER the cheap, CPU-only, leaderboard-anchored occquery (PLAN.md L39; benchmark-anchors.md:259-265 front-loads externally-verifiable topics, defers the rest).

## Central hypothesis
A correction-prioritization ranking that scores each candidate label by its ESTIMATED VALUE — how much fixing it is expected to move a downstream model — recovers more downstream task performance per unit correction budget than (a) random/uniform selection and (b) error-count / label-quality ranking (fix-the-noisiest-first), on a public 3D-detection substrate, at equal budget. The load-bearing, falsifiable core is the RELATIVE claim "value-ranked beats random at equal budget" (the headline) AND the sharper relative claim "value-ranked beats error-count-ranked at equal budget" (the one that earns the paper's title — value ≠ error count). Restated in the lab's own frame: just as occquery measures static geometry and refuses the danger verdict (PLAN.md L124-137), value-of-correction RANKS BY MODEL-MOVEMENT and explicitly refuses the assumption that the worst-labeled scene is the most valuable to fix — the entire contribution is the empirical claim that these two orderings (value vs error-magnitude) come apart, and that the value ordering is the one that recovers performance. Argument type is INDUCTIVE/empirical (measure downstream Δ on a public split, generalize to "value-ranked correction is the better budget policy"), made near-deductive by a first-principles scaffold: a label's downstream value is the influence of that label's correction on the trained model's loss on a held-out evaluation set — a quantity that is, by construction, only loosely correlated with the label's own geometric error (a large box error on a distant/rare/redundant object can move the model less than a small error on a frequent, decision-relevant one). The remove-it/counterfactual that licenses the causal reading: the ranker's score must, when you actually apply the corrections it prioritizes and retrain, produce a steeper utility-vs-budget curve than the same number of random or error-ranked corrections — high estimated value is necessary but never sufficient until the retrain confirms it.

## Sub-hypotheses
- SH1 — RANKER MECHANICS (oracle-free-ish, the smoke headline, no retraining). On a controlled substrate where you INJECT known label corruptions of known magnitude, the value-ranker's score correctly identifies and orders the injected errors better than chance, measured by noisy-label-detection F1 / point-removal AUC in the OpenDataVal style (benchmark-anchors.md:164). This validates the ranker's plumbing on data you fully control (the injected corruption IS the answer key), with NO claim about real downstream movement yet. It is the analogue of occquery's expressivity witness: a self-contained, reproducible mechanic check that needs no external oracle. CRITICAL: this tier is explicitly NOT a scientific result (benchmark-anchors.md:19) — it is the smoke test that the ranker is not broken.
- SH2 — VALUE ≠ ERROR-COUNT (the decorrelation claim; the paper's actual title). The value ranking and the error-magnitude ranking are NOT the same ordering: their rank correlation (Spearman ρ) is bounded away from 1, AND on a held-out evaluation set the value-ranked top-k corrections recover strictly more downstream metric than the error-count-ranked top-k at equal k. This is the non-identifiability-flavored core: two scenes with the SAME geometric label error but DIFFERENT downstream influence must be ranked differently by value and identically by error-count — exactly the occquery 'identical box observables, occupancy distinguishes' witness (expressivity-vs-refav.md:50-61) transposed to 'identical error magnitude, downstream-influence distinguishes'. If ρ≈1 OR the two curves coincide, the paper has no thesis.
- SH3 — DOWNSTREAM RECOVERY UNDER RETRAIN (the publishable claim; the expensive one). Retraining/fine-tuning a public detector on TRAINING data where only a value-ranked budget of labels has been corrected recovers more downstream CDS (Argoverse 2 3D-detection composite) per unit budget than (a) random selection and (b) error-count selection, on the official AV2 val split, over ≥3 seeds, reported as the FULL utility-vs-budget curve (benchmark-anchors.md:166-168), not a single budget point. This is the remove-it/counterfactual that converts the SH2 estimate into a causal claim — the corrections the ranker prioritized must, when actually applied and retrained, move the real model.
- SH4 — VALUE TRANSFERS / IS NOT DETECTOR-OVERFIT (external-validity sub-claim, optional-but-strengthening). A value ranking estimated against one model/proxy still beats random when the corrections are applied and a DIFFERENT downstream model is retrained (cross-architecture), OR the ranking estimated on a cheap proxy (small/early-stopped model, or a static influence proxy) predicts the expensive full-retrain recovery curve. This pre-empts the affirming-the-consequent failure ('it works because the ranker and the scored model are the same object') and is the value-axis analogue of dynfield's no-leakage control gate (benchmark-anchors.md:147-148).

## Success criteria (pre-registered)
- PRE-REGISTERED, ALL relative + curve-based (no movable absolute cutoff is load-bearing; benchmark-anchors.md:28-31). SMOKE (mechanic, NOT a result; SH1): on an injected-corruption substrate the value-ranker's noisy-label-detection AP/F1 and point-removal AUC beat a label-shuffled control AND a magnitude-only baseline by a pre-registered margin reported as a full removal curve — proves the plumbing, scientifically inert (benchmark-anchors.md:19,164).
- DECORRELATION (SH2; the title-earning claim): Spearman ρ between the value ranking and the error-magnitude ranking is reported with a per-scene bootstrap 95% CI and its upper CI bound is < 1 by a pre-registered margin; AND on a held-out eval set the value-ranked vs error-count-ranked utility curves separate (area-between-curves > 0 with bootstrap CI excluding 0). The RELATIVE separation is load-bearing; no absolute ρ threshold is the result.
- PUBLISHABLE (SH3; the headline scientific bar, benchmark-anchors.md:166-168): on official AV2 val, ≥3 seeds, the value-ranked utility-vs-budget curve lies ABOVE both the random curve AND the error-count curve across a pre-registered budget range (e.g. 1%..100%), with the per-budget gap's seed-variance band excluding 0 over a stated sub-range; full curve + bands released, not a single budget point; reproduction code + the value-ranking + held-out scene IDs released so others can recompute (benchmark-anchors.md:25-27).
- STRONG (adoptability, benchmark-anchors.md:169): the released benchmark — the {budget × recovery} curve protocol + the ranker + the injected-corruption oracle generator — is runnable end-to-end by a third party on the public substrate and reproduces the direction of effect on a SECOND detector or a second split (the external-validity bar in lieu of a leaderboard).
- INSTRUMENT HARDENING (always-valid even under a null, mirrors occquery's real-data payoff h3-real-data-findings.md:12-28): the act of building the injected-corruption oracle + the retrain harness surfaces and fixes ≥1 concrete defect in the valuation pipeline (e.g. a leakage path, a budget-accounting bug, a proxy that secretly re-derives error-magnitude), each documented with a commit, even if the central hypothesis is later killed.

## Kill / pivot criteria (pre-registered)
- PRIMARY KILL (benchmark-anchors.md:170): value-ranked corrections do NOT beat random at equal budget — the publishable utility-vs-budget curves for value and random are statistically indistinguishable (seed-variance bands overlap across the whole pre-registered budget range). Then 'fixing high-value labels moves the model more' is false on this substrate; report the negative as the headline (a publishable negative is a valid outcome, benchmark-anchors.md:37; asof:195-196 sets the precedent).
- TITLE KILL (SH2): value-ranking and error-count-ranking are the SAME ordering (Spearman ρ CI includes 1, OR their utility curves coincide). Then 'value ≠ error count' collapses — the paper's distinguishing claim is dead even if both beat random, because the cheaper error-count baseline already captures the signal; pivot to a pure error-count-correction paper (no value-axis novelty) or kill.
- RETRAIN-INFEASIBILITY KILL (Mac-specific, honest now): if no public detector + AV2-detection training/fine-tuning loop can be run within the available compute (NO CUDA on this arm64 Mac — verified; current repo has zero torch/detection infra — verified), then SH3 (the publishable bar) cannot be established here at all. The experiment is GATED on external GPU access; on the Mac only SH1 (injected-corruption ranker mechanics) and SH2-on-a-proxy (rank-correlation + a frozen-feature linear proxy's held-out delta, NO full retrain) are reachable. If GPU access never materializes, the publishable claim is out of reach and the stage stays at smoke/decorrelation only — state this, do not fake a retrain number.
- STUB-BASELINE KILL→REIMPLEMENT (not a kill of the idea, a kill of inheriting it): the WACV-2026 geometric box-correction code and the public VoxelNeXt checkpoint are EXTERNAL and PROVISIONAL (benchmark-anchors.md:55,159; verification flags GS-Occ3D/OccNL/ReliOcc-class repos as stub-or-unverified, benchmark-anchors.md:63-64). If either is a stub or irreproducible, it is NOT a baseline — reimplement self-contained and label it a reimplementation (benchmark-anchors.md:35-36); do NOT report against an unrunnable baseline.
- PROXY-CIRCULARITY KILL (SH4): if the only way value-ranking beats random is when the ranker and the retrained model are the SAME object (no transfer to a second architecture and no cheap-proxy→full-retrain predictivity), the result is affirming-the-consequent, not a value signal — withdraw the causal claim.

## Oracle
TWO distinct oracles, kept separate, neither allowed to be the headline (benchmark-anchors.md:25-27). (1) THE VALUE ORACLE = a self-CONTROLLED injected-corruption generator (the OpenDataVal / CAPTURe 'manufacture-the-oracle' pattern, benchmark-anchors.md:164; the gt-distrust injection pattern transposed, benchmark-anchors.md:211-213): start from the public AV2 detection labels, INJECT known label corruptions of known type+magnitude into a known subset, so the ground-truth 'which labels are wrong and by how much' is something you CONTROL, not something you trust. This makes SH1 (ranker mechanics) and the SH2 decorrelation self-contained and reproducible. RELEASE the generator code + the corrupted-subset IDs + held-out scene IDs (benchmark-anchors.md:25-27; PLAN.md:177). (2) THE DOWNSTREAM ORACLE = the OFFICIAL Argoverse 2 3D-detection metric (CDS/AP/ATE via the official devkit, benchmark-anchors.md:158) computed on the held-out val split — an EXTERNAL metric you do not define, so SH3's recovery claim is attributed to the corrections, not to a self-graded score (oracle dependence: 'downstream metric is external', benchmark-anchors.md:55,174). AVOIDING A CIRCULAR ORACLE — three structural guards, learned from h3-real-data-findings.md:6-10,87-89 (where H3 was caught grading a predicate against the SAME field it ran on): (a) the value ranking must be estimated WITHOUT touching the held-out val split the downstream metric is computed on — train/rank on TRAIN, measure on VAL, or the recovery curve is tautological; (b) the injected-corruption oracle (for SH1/SH2 mechanics) and the real-retrain oracle (for SH3 recovery) are DIFFERENT objects — never report SH3 recovery against the injected answer key, and never report SH1 detection against the retrain metric; (c) for SH4, the ranker's scoring model and the retrained downstream model must be allowed to DIFFER, so 'value' is not just 'this one model's self-influence re-derived'. The pre-registration trap to refuse (benchmark-anchors.md:172-174): do NOT inherit WACV Table-3's val-PERTURBATION numbers (the 18.7→23.0 figure) as a retrain recovery CEILING — that experiment does not exist in that paper; the real full-correction retrain delta must be MEASURED first as the achievable ceiling before any budget curve is interpreted against it.

## Baselines
- RANDOM / UNIFORM selection at equal budget — THE primary baseline; beating it is the load-bearing relative claim (benchmark-anchors.md:155-156,170).
- ERROR-COUNT / MAGNITUDE ranking ('fix the noisiest labels first') at equal budget — the naive baseline the title 'value ≠ error count' must beat (SH2); the structural analogue of occquery's box-only fairness baseline objects.distance_to_nearest_object (src/probe/predicates/objects.py:1-10).
- UNCERTAINTY-BASED selection (detector confidence/entropy) — standard active-learning competitor named at benchmark-anchors.md:160.
- GEOMETRIC-CORRECTION-WITHOUT-PRIORITIZATION (correct all labels, no ranking) — establishes the real measured full-correction recovery CEILING, NOT the inherited WACV Table-3 perturbation number (benchmark-anchors.md:166-167,172-174).
- FULL-CORRECTION and NO-CORRECTION endpoints — the ceiling and the dirty-label floor bounding every budget curve.
- STUB-BASELINE GUARD: any stub/irreproducible external baseline (WACV code, VoxelNeXt) is reimplemented self-contained and labeled a reimplementation (benchmark-anchors.md:35-36).

## Ablations
- SELECTION-POLICY ablation (the core MECE, one factor varied): hold detector + split + metric + corruption fixed, vary ONLY the selection policy {value, random, error-count, uncertainty} — each policy's utility-vs-budget curve maps to exactly one comparison (beat-random = SH3 headline; beat-error-count = SH2 title; beat-uncertainty = honest-competitor). This is the one-factor-at-a-time design the frontier prefers over a cumulative ladder.
- VALUE-ESTIMATOR ablation: vary how 'value' is computed {full-retrain influence, cheap-proxy (small/early-stopped model), static influence approximation} mapped to SH4 — reports a NULL as evidence for the cheaper estimator if a proxy matches full-retrain (the OpenVLA 'no difference → use the cheaper' pattern), and exposes proxy-circularity if only the same-model estimator works.
- TRANSFER ablation (SH4): estimate value against detector A, apply corrections, retrain detector B — isolates 'is the value signal architecture-specific?' (the no-leakage/transfer control, benchmark-anchors.md:147-148 analogue).
- CORRUPTION-TYPE ablation: vary the injected-corruption distribution {box-shift, box-scale, mislabel, drop} — checks the ranker is not tuned to one error mode; report where it HURTS (a frontier requirement: show the failure regime, not just the wins).
- BUDGET-GRID sensitivity: report the full curve and its sensitivity to the budget grid + bin choice (the movable-cutoff defense; no single budget point is the result, benchmark-anchors.md:166-168), analogous to vis-calibration's bin-sensitivity requirement (benchmark-anchors.md:245).
- SEED ablation: ≥3 seeds with variance bands on every curve — a single-seed gap is not a result.
- NOTE: NO occupancy-unknown-policy ablation here (that is occquery's geometric-mechanism ablation, PLAN.md:166-177) — value-of-correction's mechanism axis is the selection policy + value estimator, not the unknown-voxel rule; conflating them would be a category error.

## Metrics
RELATIVE-vs-ABSOLUTE framing is explicit and follows the house rule (benchmark-anchors.md:28-34): the LOAD-BEARING metrics are all RELATIVE and curve-shaped; every absolute number is a secondary internal check or an endpoint, never the result. PRIMARY (relative, load-bearing): (1) area-between-curves / per-budget gap of the value-ranked utility-vs-budget curve OVER the random curve, on official AV2 val CDS, ≥3 seeds, with seed-variance bands — beat-random is the headline (benchmark-anchors.md:155-156,166-168). (2) the same gap of value OVER error-count — the title-earning relative claim (SH2/SH3). (3) Spearman ρ(value-rank, error-magnitude-rank) with per-scene bootstrap 95% CI — a relative-separation statistic, reported as its CI not a point. SECONDARY / INTERNAL (absolute, NOT the result): downstream CDS/AP/ATE absolute values at each budget (for auditability, the per-task table the inductive frontier always pairs with relative headlines); the measured FULL-CORRECTION ceiling delta (the real one, not WACV Table-3's perturbation number, benchmark-anchors.md:172-174); SH1 noisy-label detection AP/F1 and point-removal AUC on injected corruptions (mechanic only, benchmark-anchors.md:164). CURVE/VARIANCE DISCIPLINE (benchmark-anchors.md:166-168): the utility-vs-budget relationship is reported as a FULL continuous curve over a pre-registered budget range with ≥3-seed bands — a single budget point is not a result; averaging the headline over budgets would hide where the advantage lives (the corpus's per-horizon-decay warning, applied to budget). PRE-REGISTRATION: the budget grid, the seed count, the detector, the corruption-injection distribution, and the train/val partition are all fixed and reported BEFORE running; a cutoff a reviewer can move is not a result (PLAN.md:184-186). NO CROSS-PROTOCOL COMPARISON (benchmark-anchors.md:32-34): all curves share one detector, one split, one metric devkit, one input modality — value vs random vs error-count are the SAME pipeline with only the selection policy varied (the one-factor-at-a-time discipline); any second-detector/second-split result (SH4/Strong) is labeled 'adjacent' and reported separately, never overlaid as if comparable.

## Expected results
EXPECTED (if the hypothesis holds): Spearman ρ(value, error-magnitude) is meaningfully below 1 (the orderings genuinely differ); the value-ranked utility-vs-budget curve sits ABOVE both random and error-count across the low-to-mid budget range (the regime where prioritization matters — at 100% budget all policies converge to the full-correction ceiling by construction), with the per-budget gap's ≥3-seed band excluding 0 over a stated sub-range; the steepest value advantage appears at SMALL budgets (the few highest-value corrections recover a disproportionate share of the ceiling), flattening as budget → 100%. SH1 detection-AP on injected corruptions is well above the shuffled control (plumbing sound). FALSIFYING OBSERVATIONS, precisely (each maps to a kill): (a) the value and random curves are statistically indistinguishable across the whole budget range (bands overlap everywhere) → value carries no model-movement signal → PRIMARY KILL, publishable negative (benchmark-anchors.md:170,37). (b) ρ-CI includes 1, OR the value and error-count curves coincide → value IS error-count → TITLE KILL (no distinguishing claim) even if both beat random. (c) value beats random but is indistinguishable from error-count → no value-axis novelty → pivot to an error-count-correction paper. (d) value beats random ONLY when the ranker and the retrained model are the same object (SH4 transfer fails, cheap proxy does not predict full retrain) → affirming-the-consequent → withdraw the causal claim. (e) the full-correction ceiling delta is ~0 (correcting everything barely moves CDS) → the substrate has no recoverable signal at all → the whole experiment is ill-posed on this substrate, re-scope to a noisier-label substrate (this is exactly why the real ceiling must be measured FIRST, benchmark-anchors.md:172-174, before interpreting any budget curve). HONEST CAVEAT: on this arm64 Mac (no CUDA, verified) only SH1 + SH2-on-a-proxy are observable; the SH3 retrain curve — and therefore the publishable confirm-or-falsify — is GATED on external GPU and is NOT expected to be produced on the current hardware. ## Argument chain
INDUCTIVE spine with a near-deductive scaffold and a hard counterfactual, exactly the frontier hybrid (a first-principles 'why it should work' + a remove-it test that licenses the causal reading). DEDUCTIVE scaffold (the 'why it should work', derived before any number): a label's downstream value is, by definition, the influence of correcting that label on the trained model's loss over a held-out eval set; this influence is a function of the example's frequency, redundancy, and decision-relevance, NOT of its own geometric error magnitude — therefore the value ordering and the error-magnitude ordering MUST be able to differ, and there exist labels with identical error but different influence (the SH2 non-identifiability construction, transposed from expressivity-vs-refav.md:50-61). This derivation makes the decorrelation a PREDICTION, not a hope, before the experiment runs. INDUCTIVE body: (1) SMOKE/SH1 — on injected corruptions the ranker recovers the known-wrong labels above chance (plumbing works; NOT evidence, benchmark-anchors.md:19). (2) SH2 — measure Spearman ρ(value, error-magnitude) with bootstrap CI and the held-out value-vs-error utility-curve separation; if ρ-CI excludes 1 AND the curves separate, the predicted decorrelation is confirmed. (3) SH3 — the COUNTERFACTUAL that converts estimate to cause: actually apply the value-ranked corrections, RETRAIN, and measure the external AV2 CDS recovery curve vs random and error-count; high estimated value is necessary but only the retrain makes it sufficient (RT-2-style remove-the-cause logic, applied to corrections). (4) SH4 — transfer / proxy-predictivity rules out affirming-the-consequent ('it works only because ranker==model').
