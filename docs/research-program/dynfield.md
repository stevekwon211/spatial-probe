## Motivation
"MOTIVATION + THE PRECISE GAP: occquery (Stage 1) established, on real data, that a static occupancy predicate correctly MEASURES a geometric fact (free-width = 0.8 m beside something at scene-0061) but is structurally unable to tell whether that fact MATTERS -- the 0.8 m gap beside a following lead car is benign, the identical 0.8 m gap beside a wall is not, and the difference is relative motion over time (summary.md L24-27; h3-real-data-findings.md L189-214; PLAN.md L124-137). occquery deliberately REFUSES that verdict and hands it to dynfield -- this handoff is written into the code itself in five places (freepath.py L41, reachable.py L23, queries.yaml L16-22, summary.md L6, h3-findings.md L182-183). dynfield is therefore not a new topic bolted on; it is the necessary completion of the occquery finding.\n\nTHE OPEN QUESTION dynfield answers: a planner does not consume raw sensors, it consumes a STORED spatial state. The field's reflex is to store EVERYTHING (occupancy + occupancy-flow + scene-flow + per-object velocity), but that is a representation cost with no evidence it is all load-bearing. NO public work isolates WHICH stored motion field actually changes a planner's action and IN WHICH MOMENT -- the closest substrates (nuPlan/NAVSIM closed-loop planners, PDM-Closed) measure planner QUALITY, not stored-field NECESSITY by regime (benchmark-anchors.md L126-149). The gap is a {field x regime} necessity/sufficiency matrix on a public substrate, with a static-occupancy-only baseline and a counterfactual (remove-it) measurement -- the dynamics analogue of occquery's expressivity-gap-over-box-only.\n\nWHY IT MATTERS BEYOND THE PAPER: a necessity matrix tells a real spatial-memory system (the repo's north star -- 3D as queryable/updatable STATE, PLAN.md L15) what it may DROP without changing behavior -- the parsimony prescription is directly actionable for what to store and update. It also closes the occquery loop: occquery × dynfield together is the 'is this measured situation dangerous' verdict that neither can make alone."

## Central hypothesis
CENTRAL HYPOTHESIS (one sentence, falsifiable by a single comparison): For a planner acting on a stored spatial state, the dynamics field it must store is regime-dependent and identifiable -- there exists at least one driving regime where removing a specific stored motion field (e.g. per-object velocity / occupancy-flow) measurably degrades the planner relative to a static-occupancy-only baseline, AND at least one OTHER regime where that same field is redundant (its removal leaves the planner statistically unchanged), so "which motion field, in which moment" is a real, measurable distinction rather than a uniform "always store everything" requirement.

WHY THIS IS THE RIGHT CLAIM FOR THIS REPO (the precise gap): occquery proved an occupancy predicate MEASURES a static fact (free-width = 0.8 m beside something) but established -- by its own scene-0061 lead-car finding (summary.md L24-27; h3-real-data-findings.md L189-214) -- that the SAME static measurement cannot tell a passable gap beside a following lead car from an impassable gap beside a wall, because that verdict needs relative motion over time. dynfield asks the necessity question one layer up: given that distinction requires stored dynamics, WHICH stored dynamics field, and in WHICH regime, actually changes what a downstream consumer would DO. The headline contribution is a {field x regime} necessity/sufficiency MATRIX (benchmark-anchors.md L138-140), not an accuracy number.

THE HARD HONESTY CONSTRAINT THAT SHAPES THE WHOLE PLAN (Mac / no-CUDA): The PLAN's stated dynfield anchor -- nuPlan closed-loop reactive + NAVSIM + PDM-Closed CLS-R~92 (benchmark-anchors.md L131-143) -- is a GPU-heavy closed-loop-simulator stack. Verified on this machine: arm64 Mac, torch is NOT EVEN INSTALLED, no CUDA (Bash check). A nuPlan closed-loop reactive sim with a learned planner is NOT Mac-feasible and is NOT what any code in this repo does (the repo is occupancy-retrieval over numpy grids; there is no planner, no simulator, no nuPlan adapter). Therefore the central hypothesis is split into a Mac-FEASIBLE core (counterfactual-necessity on a deterministic, analytic planner-surrogate over the Occ3D-nuScenes occupancy + flow already on disk -- pure numpy/scipy, CPU) and an explicitly-labeled GPU-GATED stretch (the nuPlan/NAVSIM closed-loop reproduction). The core is what makes the claim falsifiable on a laptop; the closed-loop reproduction is the Strong tier, deferred until a GPU box is available, and is NEVER presented as done from a Mac.

## Sub-hypotheses
- SH1 -- NECESSITY EXISTS, oracle-free headline (the analogue of occquery's H1 expressivity; no planner-quality oracle needed). There exists a constructible NON-IDENTIFIABILITY WITNESS in the spirit of tests/test_expressivity.py: two scene-pairs with IDENTICAL static occupancy at the decision frame but DIFFERENT stored motion (e.g. scene A: a 0.8 m gap beside an object closing at +3 m/s relative; scene B: identical 0.8 m gap beside an object receding at -3 m/s relative), for which any planner-surrogate that consumes ONLY static occupancy MUST emit the identical action (it sees identical input), while a surrogate that consumes the stored motion field emits different actions. This proves -- by construction, no oracle -- that a static-only stored state is INSUFFICIENT to determine the action in at least one regime, i.e. SOME motion field is necessary SOMEWHERE. This is the load-bearing, dispute-proof claim and it is fully Mac-feasible (it is a re-use of the occquery witness machinery on a motion-carrying scene). It directly formalizes the scene-0061 lead-car observation.
- SH2 -- REGIME-DEPENDENCE / SUFFICIENCY (the matrix; needs a surrogate but NOT a leaderboard). Partition decision frames into pre-registered REGIMES (defined ONLY from static + kinematic observables computable on the Mac data: {ego-following a lead vs free-flow}, {crossing/interactive agent present vs none}, {static-obstacle-only vs dynamic-agent-present}, {low vs high ego speed}). For each (field, regime) cell, measure the COUNTERFACTUAL action change when that field is ablated from the stored state vs the full-state reference. CLAIM: the matrix is non-trivial -- at least one field is NECESSARY in >=1 regime (ablating it changes the surrogate's action beyond a pre-registered, seed-bootstrapped band) AND REDUNDANT in >=1 other regime (ablating it leaves the action within the no-effect band). A uniform matrix (every field matters everywhere, or nothing matters anywhere) FALSIFIES the by-regime thesis.
- SH3 -- PARSIMONY / SUFFICIENT-SUBSET. In at least one regime there is a STRICT motion-field subset that is SUFFICIENT: a surrogate restricted to {static occupancy + that subset} reproduces the full-state surrogate's action within the no-effect band, while a strictly smaller subset does not. This is the 'a field is sufficient in some regimes' half of the PLAN claim (benchmark-anchors.md L143) and is what makes the result prescriptive ('in free-flow you may drop occupancy-flow; in interactive crossings you may not'). Reported as a sufficiency frontier per regime, not a single cutoff.
- SH4 -- HARNESS NON-LEAKAGE (the mandatory control gate; affirming-the-consequent antidote; benchmark-anchors.md L147-148). A NO-DYNAMICS control surrogate (static occupancy only, all motion fields removed) must PROVABLY collapse on an interactive regime where dynamics are definitionally required (e.g. a yield-to-crosser frame), confirming the harness is not leaking privileged motion state through a side channel (ego-pose deltas, future frames, the occupancy field implicitly encoding velocity via multi-sweep accumulation). If the no-dynamics control does NOT collapse where it must, the entire measurement is invalid (the surrogate is cheating) and SH1-SH3 results are void until the leak is closed. This is a gate, not a result -- it runs FIRST.

## Kill / pivot criteria (pre-registered)
- PRIMARY KILL (kills the by-regime thesis; the dynfield-specific falsifier, benchmark-anchors.md L144-146): a motion field is NECESSARY in EVERY regime (ablating it changes the surrogate's action beyond the no-effect band in all regimes) -- then 'which field, in which moment' collapses to 'always store it', the matrix is trivial, and the regime distinction is not a contribution. Report honestly as a negative: 'stored dynamics necessity is not regime-selective on this substrate'.
- PRIMARY KILL (kills parsimony, SH3): NO field is ever sufficient anywhere -- i.e. no strict subset of stored motion ever reproduces the full-state surrogate's action within the band in any regime -- then there is no parsimonious prescription to make. Also a publishable negative.
- WITNESS KILL (SH1): no constructible scene-pair (synthetic OR mined from real data) exists where a static-only surrogate is forced to act identically while a motion-aware surrogate differs -- i.e. the static occupancy ALREADY determines the action everywhere the data covers. Then stored dynamics add nothing the static field did not, and the entire dynfield premise fails (this would also retroactively weaken occquery's scene-0061 'danger needs dynamics' claim). Mac-detectable, and it kills early.
- HARNESS KILL / VOID (SH4): the no-dynamics control surrogate does NOT collapse on a regime where dynamics are definitionally required -- then the harness is leaking privileged motion state (via ego-pose deltas, multi-sweep occupancy implicitly encoding velocity, or future-frame peeking) and ALL necessity numbers are artifacts. This voids SH1-SH3 (does not 'kill the science' -- it kills the instrument until the leak is closed) and must be fixed before any matrix is reported.
- SUBSTRATE-VALIDITY PIVOT (Mac-specific): if the analytic planner-surrogate is so weak/degenerate that its action is dominated by hand-tuned gains rather than the stored state (i.e. the surrogate, not the field, drives the result), the Tier-1 necessity matrix is uninterpretable -> PIVOT: either (a) report only the SH1 witness + SH4 gate (the two oracle-free / control-only claims that do not depend on surrogate quality) as the Mac-feasible deliverable, OR (b) defer the matrix to the GPU Tier-2 real-planner run. Do not ship a matrix whose shape is an artifact of the surrogate.
- ANCHOR PIVOT (if Tier-2 is ever attempted): if the nuPlan/NAVSIM metric/split cannot be primary-confirmed or PDM-Closed cannot be reproduced to its published number on the available GPU, the closed-loop anchor is dropped and the result is reported as a DEFINE-tier benchmark (no leaderboard placement), per the same discipline occquery applies to its H2 EvalAI pre-req (PLAN.md L152-154).

## Oracle
"GROUND-TRUTH / ORACLE CONSTRUCTION (and how the circular-oracle trap is avoided): dynfield is unusual in that its PRIMARY claims need NO quality oracle -- they are counterfactual and control-based, which is the strongest position in the lab's epistemics (benchmark-anchors.md L25-27, 'self-oracle is never the headline; lead with the part that needs none'). Specifically: (1) SH1 (necessity witness) needs no oracle at all -- it is a non-identifiability CONSTRUCTION (identical input MUST yield identical output), exactly like tests/test_expressivity.py; the 'truth' is logical, not measured. (2) SH4 (harness non-leakage) needs no quality oracle -- it is a CONTROL: a surrogate provably must fail where dynamics are required, and observing it does is the check. (3) SH2/SH3 (the matrix) compare each ablated surrogate to the FULL-STATE surrogate as the SELF-REFERENCE -- the 'oracle' is the planner's own action under complete information, so there is no external label to dispute; the measured quantity is a DELTA between two configurations of the same deterministic surrogate, not agreement with a hand-drawn answer key.\n\nWHERE STORED MOTION FIELDS COME FROM (the data-construction the matrix consumes), and the circular-oracle guard: the stored dynamics fields being ablated must come from a source INDEPENDENT of the surrogate that consumes them, or the test is tautological. Per-object velocity is taken from nuScenes sample_annotation velocity (the dataset's own tracked-box motion, NOT re-derived from the occupancy the surrogate also reads); occupancy-flow / scene-flow, if used, is taken from a RELEASED third-party flow field or computed from multi-frame GT occupancy by a fixed, released procedure -- and CRITICALLY, the regime LABELS (following / interactive / static-only) must be derived from a source the surrogate does not also use as its action input, to avoid 'the field that defines the regime is trivially the field that matters in it'. The Occ3D dense-GT occupancy (mask=None path, occ3d.py L43-64, ~0 unknown) is the static-state oracle, reused unchanged from occquery; this is the same released, recomputable substrate, and its construction code + held-out scene IDs are released (benchmark-anchors.md L27).\n\nCIRCULAR-ORACLE AVOIDANCE (explicit, mirroring gt-distrust dropping its camera-mask leg, benchmark-anchors.md L216-225): (a) the ablated field must NOT be the same signal the surrogate's regime partition is cut on (else necessity is definitional); (b) per-object velocity must come from the dataset's tracking, not from differencing the very occupancy the surrogate reads (else 'velocity matters' just restates 'the occupancy moved'); (c) the full-state reference and the ablated surrogate must differ in EXACTLY the stored field under test and nothing else (one-factor isolation). Any leg where the 'truth' shares the surrogate's own input is dropped, not reported.\n\nHARD DEPENDENCY FLAGGED: the real Occ3D adapter currently returns objects=() (occ3d.py L13, L106) -- so TrackedBox.velocity (scene.py L37), the most natural per-object stored motion field, is NOT populated from real data yet. Populating it requires nuScenes-devkit box loading (the optional [data] dep, pyproject), which IS pip-installable on Mac (CPU) but is presently out of the adapter's scope. This is the first concrete build step and a hard upstream dependency on extending the occquery-era adapter."

## Baselines
- STATIC-OCCUPANCY-ONLY baseline (the load-bearing naive baseline -- the dynfield analogue of occquery's box-only RefAV baseline): a planner-surrogate that consumes ONLY the static Occ3D occupancy field (no motion field whatsoever), reusing the occquery reachable_free_field substrate (reachable.py) unchanged. The central necessity claim is measured RELATIVE to this baseline: a field is 'necessary in a regime' iff adding it to this static-only state changes the surrogate's action beyond the no-effect band. This is the direct counterpart to 'free-space predicates beat the best box-only approximation by >=20 F1' (PLAN.md L160) -- the load-bearing claim is the RELATIVE gap over the static-only baseline, never an absolute action-quality score.
- NO-DYNAMICS CONTROL (SH4 gate): identical to the static-only baseline but used specifically as the harness-leakage control -- it MUST collapse where dynamics are required. Distinct role from the baseline above (one is the scientific reference for the matrix; this is the validity gate).
- MOTION-FIELD-SHUFFLED CONTROL (the affirming-the-consequent antidote, borrowed from gt-distrust's shuffled-control, benchmark-anchors.md L217): a surrogate fed a PERMUTED/randomized version of the motion field (correct field, wrong scene assignment). If ablating the TRUE field and ablating a SHUFFLED field change the action equally, then the surrogate is not actually USING the field's information content and the 'necessity' is spurious. The true field must beat the shuffled control.
- PER-FIELD ablation arms as mutual baselines (MECE, see ablations): {static-only} vs {static + per-object velocity} vs {static + occupancy-flow} vs {static + scene-flow} vs {static + full motion}, each a baseline for the others within a regime, so necessity/sufficiency is read off the lattice.
- EXTERNAL ANCHOR baseline (GPU-gated, Tier 2 only): PDM-Closed (released autonomousvision/tuplan_garage, benchmark-anchors.md L132) reproduced to its published nuPlan number as the harness-validity anchor. NOT a Mac baseline; reimplement-and-label discipline applies if the public code is a stub (benchmark-anchors.md L35-36).

## Ablations
- MECE ABLATION 1 -- FIELD ABLATION (one factor = which stored motion field is present; maps 1:1 to SH2 necessity). The lattice: {static-only} -> {+per-object velocity} -> {+occupancy-flow} -> {+scene-flow} -> {+full motion}, each field added/removed in isolation against the same surrogate. Each arm maps to exactly one sub-claim cell. Report NULLS as evidence (a field that never changes the action anywhere is a positive parsimony finding, benchmark-anchors.md-style 'report nulls', not a buried negative). Show where a field HURTS if it does (e.g. a noisy flow field degrading an action a clean static read got right) -- the ladder-style monotone 'everything helps' is explicitly avoided.
- MECE ABLATION 2 -- REGIME PARTITION (one factor = regime; maps 1:1 to SH2 by-regime). Cross every field-arm with every pre-registered regime {following vs free-flow} x {interactive-agent vs none} x {static-obstacle-only vs dynamic-agent} x {low vs high speed}. The CROSS (field x regime) is the matrix; the off-diagonal contrast (necessary-here, redundant-there) is the result. This is the factorial the occupancy-survey warns is missing when people only ladder -- here it is required.
- MECE ABLATION 3 -- HARNESS-LEAKAGE GATE (SH4): toggle the suspected leak channels one at a time -- (a) ego-pose-delta available vs zeroed, (b) multi-sweep accumulated occupancy (which implicitly encodes motion) vs single-sweep, (c) future-frame access vs strictly-causal. Each isolates one way the static-only control could cheat. The control must collapse ONLY when a genuine motion field is added, not when a leak is opened.
- MECE ABLATION 4 -- FIELD-FIDELITY (one factor = the stored field's quality, isolating SH2's robustness): full-resolution field vs the SHUFFLED control vs a quantized/down-sampled field. Splits 'the field's INFORMATION is necessary' from 'the surrogate merely reacts to the field's presence'. The true field must beat the shuffled control (the affirming-the-consequent counterfactual).
- MECE ABLATION 5 (GPU-gated, Tier 2 only) -- SURROGATE-vs-REAL-PLANNER (one factor = analytic surrogate vs PDM-Closed-class learned planner). Re-run the same field x regime ablation against the real planner in closed loop; the claim is the matrix's qualitative shape is invariant to this swap. Reported separately, 'adjacent not comparable' to the Mac matrix. ALSO (per the RefAV lesson, if any LLM/model-in-the-loop variant of the surrogate is ever used) report a program-failure / surrogate-degeneracy rate alongside the action metric.
- ABLATION 6 -- UNKNOWN-POLICY 3-RULE SENSITIVITY (inherited from occquery, reused unchanged): re-run the matrix under UnknownPolicy {free, occupied, ignored} on the unobserved voxels of the OBSERVED (masked) view. The necessity contrast must be STABLE across the 3 rules, or scope shrinks to the dense-GT view (the same falsification shape occquery uses, PLAN.md L171-177; benchmark-anchors.md L108-110). This is where the masked-view conditioning variable from the adapter (occ3d.py L52-56) becomes the test.

## Metrics
"PRIMARY METRIC (oracle-free, headline): the SH1 WITNESS holds (binary, by construction) -- count of constructible static-identical / motion-differing scene-pairs on which the static-only surrogate is provably forced to act identically while the motion-aware surrogate differs (synthetic + >=1 mined from real data). Like occquery's expressivity coverage, this is a SEPARATION count, not a score.\n\nPRIMARY METRIC (the matrix, relative): per (field, regime) cell, the COUNTERFACTUAL ACTION CHANGE = a distance between the surrogate's action trajectory WITH vs WITHOUT the stored field (e.g. L2 over the planned longitudinal+lateral control sequence, or a decision-flip indicator yield/proceed), reported as a FULL DISTRIBUTION with per-scene bootstrap 95% CIs and seed-variance bands (benchmark-anchors.md L138-140), NOT a single number. NECESSITY = the cell's action-change CI lies above the pre-registered no-effect band; REDUNDANCY = the CI lies inside the band. SUFFICIENCY (SH3) = a subset's action matches full-state within the band.\n\nRELATIVE-vs-ABSOLUTE FRAMING (the lab's hard rule, benchmark-anchors.md L28-31): the LOAD-BEARING claim is the RELATIVE, WITHIN-MATRIX CONTRAST -- the SAME field is necessary in regime X (CI above band) and redundant in regime Y (CI in band). The absolute action-change magnitude is a SECONDARY internal-validity check only; a reviewer can move the no-effect band and the headline (necessary-here-not-there) must survive because it is a contrast between cells measured under the IDENTICAL band. No absolute 'the field adds N units of quality' claim is headlined. NO cross-protocol comparison (benchmark-anchors.md L32-34): the analytic-surrogate matrix and the GPU closed-loop matrix are DIFFERENT protocols and are reported as 'adjacent, not comparable' -- the Tier-2 claim is only that the QUALITATIVE SHAPE recurs, never that the numbers match.\n\nCONTROL METRICS: SH4 leakage gate (no-dynamics control action-collapse magnitude on a required-dynamics regime -- must be large); shuffled-field control (true-field action-change must exceed shuffled-field action-change, with CI separation). Pre-registered cutoffs for every band, reported as curves over the band parameter."

## Scope — in
- IN-CLAIM: stored-motion-field NECESSITY and SUFFICIENCY are REGIME-DEPENDENT and MEASURABLE -- there is at least one regime where a specific stored motion field changes a planner-surrogate's action and at least one where it does not (the {field x regime} matrix).
- IN-CLAIM: a static-occupancy-only stored state is INSUFFICIENT to determine the action in at least one regime (the SH1 witness; the formal statement of occquery's scene-0061 lead-car-vs-wall observation).
- IN-CLAIM: the harness does not leak privileged motion state (SH4 control passes) -- a precondition the paper proves, not assumes.
- IN-CLAIM (prescriptive, the contribution's payoff): a per-regime sufficient-subset prescription -- which stored motion fields a planner consumer may drop in which regime without changing its action -- released as a benchmark (matrix + ablation harness + held-out IDs).
- IN-SCOPE substrate (Mac-feasible): Occ3D-nuScenes occupancy (already on disk) + nuScenes box velocities (via nuscenes-devkit, CPU) + a DETERMINISTIC ANALYTIC planner-surrogate (numpy/scipy). Regimes defined from static+kinematic observables.
- IN-SCOPE reuse: the occquery core unchanged -- raycast/DDA, OccupancyGrid, reachable_free_field, the Occ3D adapter, the UnknownPolicy 3-rule machinery, and the test_expressivity witness pattern (this is the 'extract on second use' payoff: dynfield is the second consumer of the reachable-field + adapter substrate).

## Scope — out (what it does NOT claim)
- OUT-OF-CLAIM: dynfield does NOT claim to produce a state-of-the-art PLANNER or to improve closed-loop driving score -- it is a NECESSITY DIAGNOSTIC on stored representation, not a planning method (mirrors occquery's 'MEASURES, does not judge'; the lane is diagnosis-by-gap-size, not engineering-by-beat-SOTA).
- OUT-OF-CLAIM: it does NOT assert which motion-field ESTIMATOR (FB-OCC, a flow network, a tracker) is best, nor any perception-quality number -- the stored fields are taken as given (from the dataset / a released field), and error is attributed to the FIELD'S PRESENCE/ABSENCE, not its estimation quality.

- **major**: BAND IS A SINGLE SHARED SCALAR ACROSS HETEROGENEOUS REGIMES — a hidden MECE violation in the comparison protocol. The 'no-effect band' is pre-registered as one width and the load-bearing contrast is 'same field, CI above band in regime X, CI inside band in regime Y, measured under the IDENTICAL band' (metrics block). A single absolute band then mechanically yields 'necessary in high-speed, redundant in low-speed' as an ARTIFACT of metric scale, not field necessity — manufacturing the headline contrast from a units problem. The plan's claim that 'a reviewer can move the band and the contrast survives' is false if the contrast is scale-induced: moving the band shifts both cells together only if their noise scales identically, which across regimes it does not.
  - fix: Either (a) normalize the action-change metric per-regime to a scale-free quantity (e.g. fraction of the regime's own achievable control range, or a decision-FLIP indicator yield/proceed which is dimensionless and regime-comparable) so one band is legitimately shared; or (b) pre-register a PER-REGIME band derived from that regime's own null/shuffled-control distribution, and make the load-bearing claim the relative position of TRUE-field vs SHUFFLED-control WITHIN each regime (already a listed control) rather than vs a cross-regime absolute band. - **minor**: BASELINE / ABLATION OVERLAP — the no-dynamics control (SH4 gate) and the static-occupancy-only baseline are described as 'distinct roles' but are the SAME artifact (static occupancy, all motion removed), and the multi-sweep-occupancy leak channel undercuts BOTH simultaneously. ### logic (inductive/deductive soundness — adversarial top-venue review) lens — verdict: revise — the plan is unusually self-aware about fallacies (it names all four and builds controls), and its repo claims all verify (adapter objects=(), velocity unpopulated, witness pattern in test_expressivity.py, scene-0061 lead-car finding, benchmark-anchors epistemics — all confirmed against source). But three logical defects are load-bearing and currently let the headline claim survive regardless of outcome, which is the signature of an unfalsifiable design. Not reject-as-unsound: the SH4 gate + shuffled control + counterfactual-ablation are genuinely the right causal instruments, and the kills are mostly real. It needs revision to (a) stop calling the tautological SH1 witness a "dispute-proof load-bearing claim," (b) close the regime/field common-cause circularity, and (c) remove the escape hatch that makes the SH2 kill non-binding. Fix those and it is an honest DEFINE-tier benchmark; ship as-is and a reviewer kills it on "you proved a tautology and made your own kill unreachable."
- **blocker**: UNFALSIFIABLE CENTRAL CLAIM via an escapable kill (heads-I-win structure). The plan's own honest prior says the most likely outcome is velocity matters 'almost everywhere' (a twitchy analytic surrogate brakes for any closing object) -> near-uniform matrix -> this should trigger the PRIMARY KILL ('necessity is uniform, not regime-selective'). But the SUBSTRATE-VALIDITY PIVOT says: if the matrix shape 'tracks the surrogate gains rather than the field,' declare the surrogate degenerate and retreat to reporting only SH1+SH4. The kill is escapable; the central hypothesis ('which field, in which moment, is real and measurable') is therefore not falsifiable on the Mac-feasible tier — every adverse result routes to 'surrogate problem, defer to GPU,' never to 'thesis wrong.' This is the textbook unfalsifiable-claim defect a top venue rejects on.
  - fix: Make the surrogate-degeneracy verdict PRE-REGISTERED and INDEPENDENT of the matrix outcome. Before running the ablation, commit a surrogate-validity criterion measured on a held-out probe set (e.g. the surrogate's action must vary with a controlled input it SHOULD respond to AND be invariant to one it should NOT, with thresholds fixed in advance). Only if the surrogate fails that pre-committed probe may you invoke the pivot. - **blocker**: CIRCULAR ORACLE one level up — the regime partition shares a common cause with the ablated field's relevance, making 'redundant in the static-only regime' definitional, not empirical. The guard ('the ablated field must not be the signal the regime is cut on') is too narrow. The regimes are {dynamic-agent-present vs static-obstacle-only}, {interactive-agent vs none}, {following vs free-flow}. The 'static-obstacle-only / no-agent' regime is DEFINED by the absence of moving agents — so a per-object motion field is trivially redundant there (there is no object whose velocity could matter). The headline contrast 'velocity necessary in dynamic-agent regime, redundant in static-only regime' then restates the regime definition rather than discovering a fact about stored representation. Common cause (presence of moving agents) drives both the regime label and whether the motion field can carry information — correlation-as-structure dressed as a finding.
  - fix: The non-trivial, non-circular contrast must hold WITHIN a regime that contains moving agents in BOTH arms — e.g. 'per-object velocity is necessary in interactive-crossing frames but REDUNDANT in following frames,' where both arms have a moving agent but the field's decision-relevance differs.
